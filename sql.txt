No, you can't use aggregate functions directly in the WHERE clause — because WHERE filters rows before aggregation happens.

Instead, use aggregate functions in the HAVING clause, which filters after aggregation.

         Filtering and Sorting
WHERE: Filter rows
ORDER BY: Sort results
DISTINCT: Remove duplicates
LIMIT: Restrict number of results


🧠 SQL Query Execution Order (Step-by-Step)
Step	Clause	What It Does
  1️⃣    	FROM	Chooses the table(s) or joins them
2️                 WHERE	Filters rows based on condition
3️⃣	GROUP BY	Groups rows (for aggregation)
4️⃣	HAVING	Filters groups (after grouping)
5️⃣	SELECT	Chooses which columns or expressions to return
6️⃣	DISTINCT	Removes duplicate rows from selected results
7️⃣	ORDER BY	Sorts the result rows
8️⃣	LIMIT	Returns only a limited number of rows


Filtering happens first, selection and sorting come later.

┌────────────────────┐
│   1. FROM          │  ← Start by loading data from the table(s)
├────────────────────┤
│   2. WHERE         │  ← Filter rows based on a condition
├────────────────────┤
│   3. GROUP BY      │  ← Group rows if using aggregate functions
├────────────────────┤
│   4. HAVING        │  ← Filter groups created by GROUP BY
├────────────────────┤
│   5. SELECT        │  ← Choose which columns or expressions to return
├────────────────────┤
│   6. DISTINCT      │  ← Remove duplicates if needed
├────────────────────┤
│   7. ORDER BY      │  ← Sort the final result
├────────────────────┤
│   8. LIMIT         │  ← Take only a portion of the sorted result
└────────────────────┘
